# Database Roadmap

The RISE database will grow gradually as the marketplace matures.

## Planned Core Tables

- `users`
- `roles`
- `permissions`
- `countries`
- `cities`
- `categories`
- `stores`
- `products`
- `services`
- `orders`
- `payments`
- `reviews`
- `favorites`
- `notifications`
- `messages`
- `trust_profiles`
- `trust_events`
- `audit_logs`

## Later Marketplace Tables

- `product_images`
- `store_images`
- `store_documents`
- `store_hours`

## Principle

Add tables only when the current sprint requires them.

The schema should stay normalized enough to avoid future rework, but not overcomplicated before private alpha.
